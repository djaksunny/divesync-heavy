# DiveSync Heavy — Buoyancy Control & Data Acquisition System

## Overview

DiveSync Heavy is a buoyancy control and data acquisition system developed for underwater robotics experiments.  
It integrates embedded firmware, pressure sensing, actuator control, and Python-based logging and analysis tools for research on depth control strategies (MPC, human-in-the-loop control, and learning-based approaches).

The system is designed for reproducible experiments and dataset generation for control research.

---

## System Architecture

```

KiCad (hardware design)
↓
Firmware (ESP32 / microcontroller control loop)
↓
Serial telemetry (CSV stream)
↓
Python Logger (data acquisition + experiment control)
↓
Raw Dataset Storage
↓
Processing / Verification
↓
Analysis / ML / Control experiments

```

## Data Format

### Raw Telemetry (CSV)

Generated by firmware over serial:

```

time_ms, actuator_raw, motor_cmd, depth_m, pressure_mbar, temp_c, depth_ok, battery_v

````

### Metadata (JSON)

Each experiment includes:

- experiment ID
- control mode (manual / MPC / RL)
- duration (requested and actual)
- start timestamp
- optional notes

Example:

```json
{
  "experiment-id": "20260625-201530",
  "control-mode": "manual",
  "duration": 30,
  "actual-duration": 29.8,
  "start-time": 1750880000,
  "notes": "ballasted test run"
}
````

## Workflow

### 1. Hardware Setup

* Design PCB in KiCad
* Deploy firmware to microcontroller
* Connect pressure sensor + actuator system

---

### 2. Data Collection

Run:

```bash
python logger/logger.py
```

Steps:

* select COM port
* configure experiment parameters
* wait for system initialization
* collect telemetry

Outputs:

* raw.csv
* boot.txt
* metadata.json

---

### 3. Data Processing

Run:

```bash
python analysis/verify.py
```

Generates:

* cleaned dataset
* velocity estimates
* spike filtering
* feature extraction

---

### 4. Visualization

```bash
python analysis/plot_run.py
```

Produces:

* depth vs time plots
* actuator response plots
* control performance metrics

---

### 5. Control Modes

Supported experiment modes:

* manual (human-in-the-loop PWM control)
* mpc (baseline controller)
* rl (learning-based controller)

---

## Research Goals

* Evaluate buoyancy control performance under different control strategies
* Generate dataset for learning-based depth control
* Compare human control vs classical MPC vs learned policies
* Develop robust depth control under varying payload conditions

---

## Safety Notes

* Always ensure emergency stop capability (Ctrl+C or firmware stop command)
* Test with low buoyancy / safe conditions first
* Do not run autonomous control without supervision
